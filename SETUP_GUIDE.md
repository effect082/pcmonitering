# PC ON/OFF 모니터링 상세 설정 가이드 (기관용)

이 가이드는 `PC ON/OFF 모니터링 시스템`을 귀하의 기관에 맞게 설정하고 직원들에게 배포하는 전체 과정을 상세히 설명합니다.

---

## 1단계: Google Apps Script 백엔드 설정

앱이 작동하기 위한 데이터베이스 역할을 할 Google 시트를 설정합니다.

### 1-1. 시트 생성
1. [Google Drive](https://drive.google.com) 에 접속 후 구글 계정으로 로그인합니다.
2. 왼쪽 상단의 `[ + 새로 만들기 ]` → `[ Google 스프레드시트 ]` 를 클릭합니다.
3. 생성된 문서의 제목을 `(기관명) 근태 모니터링 DB` 등 구별하기 쉬운 이름으로 변경합니다.

### 1-2. 코드 붙여넣기
1. 상단 메뉴에서 `[확장 프로그램]` → `[Apps Script]`를 클릭합니다.
2. 새 탭으로 열리는 Apps Script 에디터에 기본으로 작성되어 있는 `function myFunction() { ... }` 코드를 모두 지웁니다.
3. 이 저장소의 `backend/Code.gs` 파일 내용을 복사하여 빈 에디터에 붙여넣습니다.
4. 상단 툴바의 디스크 모양 아이콘 `[저장]` 을 클릭합니다. (또는 Ctrl+S / Cmd+S)

### 1-3. 웹 앱으로 배포하기 ⭐️중요⭐️
1. 우측 상단의 파란색 `[배포]` 버튼 클릭 → `[새 배포]` 를 선택합니다.
2. '유형 선택' 옆의 톱니바퀴 아이콘 ⚙️ 을 클릭하고 `[웹 앱]` 을 선택합니다.
3. 아래 정보를 정확히 설정합니다:
   - **설명**: "PC ON/OFF 모니터링 서버 v1" (원하는 이름 입력)
   - **다음 사용자로서 실행**: `나 (자신의 이메일 주소)`
   - **액세스 권한이 있는 사용자**: `모든 사용자` (반드시 '모든 사용자'를 선택해야 합니다. 그렇지 않으면 직원이 로그인할 수 없습니다.)
4. `[배포]` 버튼을 클릭합니다.
5. "액세스 승인" 창이 뜨면 `[액세스 승인]`을 누릅니다.
6. 계정을 선택하고, "Google에서 확인하지 않은 앱" 경고가 나오면 좌측 하단의 `[고급]` → `[~로 이동(안전하지 않음)]` 을 클릭합니다.
7. `[허용]` 을 클릭하여 권한을 부여합니다.
8. 완료 화면이 나타나면 `웹 앱 URL` 항목 아래의 주소를 복사합니다. (예: `https://script.google.com/macros/s/.../exec`)

## 2단계: 프론트엔드 호스팅 설정 (GitHub Pages)

### 2-1. 저장소 Fork
1. 현재 이 GitHub 저장소 페이지의 우측 상단 `[Fork]` 버튼을 눌러 본인의 계정으로 복사해 갑니다.

### 2-2. 배포 설정
1. 복사한 본인 계정의 저장소로 이동합니다.
2. 상단 탭에서 `[Settings]` 를 클릭합니다.
3. 좌측 메뉴에서 `[Pages]` 를 클릭합니다.
4. "Build and deployment" 섹션에서 Source를 `Deploy from a branch`로 설정합니다.
5. "Branch" 아래의 드롭다운을 클릭하여 `main`을 선택하고, 폴더는 `/ (root)` 상태로 둔 뒤 `[Save]` 를 클릭합니다.
6. 1~2분 정도 기다린 후 브라우저를 새로고침하면 상단에 활성화된 주소가 나타납니다. (예: `https://아이디.github.io/pcmonitering/`)

---

## 3단계: 관리자 대시보드 서버 연동

이제 생성한 서버(GAS URL)를 웹 앱에 연결할 차례입니다.

### 3-1. 관리자 페이지 로그인
1. 2단계에서 얻은 GitHub Pages 주소로 접속합니다.
2. 하단의 `[관리자 로그인]` 을 클릭합니다.
3. 초기 비밀번호인 `1234`를 입력하여 로그인합니다.

### 3-2. 기관 설정하기
1. 상단 탭에서 `[서버 설정]` 을 클릭합니다.
2. **GAS 웹앱 URL**: 1-3단계에서 복사해 둔 긴 주소를 붙여넣고 `[저장]` 을 누릅니다.
3. `[연동 테스트]` 버튼을 눌러 "✅ 연동 성공" 메시지가 뜨는지 확인합니다.
4. **기관명**: 기관 이름을 입력하고 `[저장]` 을 누릅니다.
5. (선택) **직인 업로드**: 인쇄물에 들어갈 직인 파일(.png, .jpg)을 선택합니다.
6. 우측 상단의 `[비밀번호 변경]` 을 눌러 관리자용 초기 비밀번호(1234)를 반드시 다른 번호(예: 숫자 4자리)로 변경하세요.

---

## 4단계: 직원에게 배포하기

관리자 대시보드 설정이 완료되면, 직원들이 접속할 수 있는 전용 URL이 생성됩니다.

### 4-1. 배포 링크 공유
1. `[서버 설정]` 탭 하단의 "직원 배포 링크" 섹션으로 이동합니다.
2. `[📋 링크 복사]` 버튼을 눌러 복사한 뒤 카카오톡, 사내 게시판, 이메일 등으로 직원들에게 전달합니다.
   - 모바일 사용자를 위해 표시된 QR코드를 인쇄하여 벽에 붙여두어도 됩니다.
3. 직원은 전달받은 링크로 접속하면 자동으로 서버가 연동되며, 바로 `[회원가입]` 후 출퇴근 기록이 가능합니다.

---

## 5단계: 데스크톱 앱 배포 및 설정 (선택사항)

PC를 켜고 끌 때 자동으로 출퇴근 시간을 기록하려면 데스크톱 전용 앱을 설치해야 합니다.

1. GitHub 저장소의 `Actions` 탭으로 이동합니다.
2. "Build Electron App" 워크플로우를 클릭하고 가장 최근 성공한 빌드를 선택합니다.
3. 하단 "Artifacts"에서 `Setup-PC-Monitor-App` 을 클릭하여 설치 파일(.exe)을 다운로드 받습니다.
4. 다운로드 받은 파일을 직원들의 PC에 설치합니다. (또는 인트라넷 파일 서버에 올려두고 다운받아 설치하도록 안내)
5. 설치 후 앱이 실행되면 **"이름"**을 입력하는 창이 뜹니다.
6. 직원의 이름(웹에서 회원가입한 이름과 동일하게)을 입력하고 `[저장하고 숨기기]`를 누릅니다.
7. 이후 PC가 부팅되거나 종료될 때마다 백그라운드에서 자동으로 서버와 통신하여 출퇴근 시간을 기록합니다.
